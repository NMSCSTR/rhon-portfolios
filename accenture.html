    echo "Adult";
} else {
    echo "Minor";

// Repeating iteration loops
for ($i = 0; $i < 3; $i++) {
    echo $i; // Outputs 012

4. Scope & Pass By Value vs Reference

<?php
// Pass By Value (Default copy instantiation)
function addValue($num) { $num += 5; }
$x = 10; addValue($x); // $x is still 10

// Pass By Reference (Uses & pointer address modification)
function addRef(&$num) { $num += 5; }
$y = 10; addRef($y); // $y becomes 15
?>

5. Recursion

Recursion is a process where a function executions calls to itself. It requires a rigid base case condition to avoid runtime execution stack overflows.

<?php
function countDown($n) {
    if ($n <= 0) { echo "Stop!"; return; } // Base Case
    echo $n . " ";
    countDown($n - 1); // Recursive Call
}
countDown(3); // Outputs: 3 2 1 Stop!
?>

Part 3 – Object-Oriented Programming (OOP)

OOP is a development approach mapping real-world components to architectural code structures.

Core Concepts Defined

  • Class: The program design template or structural blueprint.
  • Object: A concrete single instance of a class loaded into active memory.
  • Encapsulation: Protecting inner data states using private/protected visibility fields, forcing access via public getter/setter methods.
  • Inheritance: Reusing baseline code structures by letting child classes extend a parent class.
  • Polymorphism: Allowing distinct classes to use identical method names while writing unique functional outputs (many forms).
  • Abstraction: Exposing critical interface layouts while hiding deep backend structural mechanics.

Code Implementation Blueprint

<?php
// Parent Abstract Class (Abstraction)
abstract class Animal {
    protected $name; // Encapsulation

    public function __construct($name) { $this->name = $name; }
    abstract public function speak(); // Forces child customization
}

// Child Inheritance execution
class Dog extends Animal {
    public function speak() {
        return $this->name . " says: Bark!"; // Polymorphism implementation
    }
}

$pet = new Dog("Buddy"); // Object instantiation
echo $pet->speak(); // Outputs: Buddy says: Bark!
?>
What is the difference between Interfaces and Abstract Classes?

Interfaces function as a rigid behavior contract containing only empty structural definitions; a class can implement several interfaces. Abstract Classes act as structural parent baselines that mix both normal built code methods and empty templates; a class can inherit from only one parent.

SOLID Principles

  • S - Single Responsibility: A single class does exactly one targeted job.
  • O - Open/Closed: Programs are open to feature scaling but closed to core modifications.
  • L - Liskov Substitution: Child structures must seamlessly replace parent positions without breaking application workflows.
  • I - Interface Segregation: Splitting massive, bulky contracts into tiny targeted interfaces.
  • D - Dependency Inversion: Code structures should look up abstract templates rather than concrete classes.

Part 4 – SQL Masterclass

Relational databases require clear structural queries to store, access, and manage information sets.

1. Query Structuring Clauses

-- Retrieving and sorting filtered table records
SELECT employee_id, salary 
FROM employees 
WHERE salary >= 25000 
ORDER BY salary DESC;

2. Aggregate Calculations & Aggregation Grouping

-- Aggregating logs and testing groups using HAVING
SELECT department_id, COUNT(id) AS staff_count, AVG(salary) AS avg_income
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 30000;

3. Structural Table Joins

  • INNER JOIN: Extracts only overlapping records present in both tables.
  • LEFT JOIN: Pulls all rows out of the left table alongside matching rows out of the right side table. Missing values display as NULL.
SELECT users.name, orders.order_id 
FROM users 
LEFT JOIN orders ON users.id = orders.user_id;

4. Database Normalization Models

  • 1NF: Table data cells must be atomic, with only single individual data elements.
  • 2NF: Achieves 1NF, and eliminates partial dependencies where non-key columns rely on only a piece of a composite key.
  • 3NF: Achieves 2NF, and completely eliminates transitive column dependencies (non-key columns relying on other non-key columns).

5. Transaction Control Execution

START TRANSACTION;
UPDATE accounts SET balance = balance - 1000 WHERE id = 1;
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;
COMMIT; -- Complete operations cleanly if safe
-- ROLLBACK; -- Revert everything back if error strikes

Part 6 – REST API Concepts

REST APIs act as standardized communication endpoints between isolated applications or architectural components.

1. HTTP Request Methods

2. Critical Status Feedback Signals

  • 200 OK: Request succeeded with data.
  • 201 Created: Request succeeded and a record was added.
  • 400 Bad Request: Client sent bad data or missing parameters.
  • 401 Unauthorized: Missing identity tokens or needs active login state.
  • 403 Forbidden: Identity known, but user lacks necessary authorization permissions.
  • 404 Not Found: The requested endpoint or resource does not exist.
  • 500 Server Error: The application server encountered an internal crash.
Architecture Tools: REST vs SOAP

REST is an architectural approach built around flexible data payloads like JSON, prioritizing execution speed and efficiency. SOAP is a highly strict messaging protocol running exclusively on XML, heavily utilized by large enterprise networks due to its rigid built-in security features.

Part 7 – MVC Architecture

MVC isolates application concerns to make code modular, highly maintainable, and scalable.

The Architectural Components

  • Model: Directs the application data structures, manages business rules, and interfaces with the database.
  • View: Formats and displays the interface design onto the user's screen (HTML/Blade templates).
  • Controller: Functions as the operational processor, translating incoming client inputs into database directions and updating the visual view (the brain).

Laravel Core Lifecycle Flow

User Request → public/index.php Entry → Middleware Filters → Route Mapping → Controller Logic Execution → Model Database Call → Blade Template Generation → Client View Response.

What is a "Fat Controller" anomaly?

"A fat controller happens when you pack too much database logic or calculations inside a controller method. We avoid this by keeping controllers thin, meaning they should only delegate tasks. Complex business logic should be pushed down into the Model or custom Service layers."

Part 8 – Git & Version Control Operations

Git tracks code history across local files, protecting code bases from data loss during team collaboration phases.

1. Essential Terminal Commands

  • git init: Initializes a brand new local repository.
  • git status: Inspects current project modifications and tracking state.
  • git add .: Stages all active changes for tracking.
  • git commit -m "msg": Saves the staged snapshot into the local history log.
  • git checkout -b branch_name: Spreads out a new feature development path and switches to it.
  • git pull origin main: Fetches the newest updates from the remote cloud repository and merges them into your active files right away.
  • git push origin branch_name: Uploads local branch changes to GitHub.

2. Resolving Merge Conflict Errors

Merge conflicts occur when two developers write changes to the exact same line of code within a file, and Git cannot determine which one to keep automatically.

Part 10 – Data Structures

Data structures organize machine data collections to ensure system execution memory remains clean and performant.

Core Implementations

  • Arrays: Sequential memory slots holding values accessible via automated index locations.
  • Linked Lists: Elements separated across system memory links, referencing each other using structural node pointers.
  • Stack (LIFO): Sequential data queue where items enter and exit through the same top slot—Last In, First Out.
  • Queue (FIFO): Standard line management system where components enter through the back and leave through the front—First In, First Out.
  • Hash Table: Maps specific data tracking strings to targeted values using hashing logic for near-instant access.
  • Binary Search Tree (BST): Sorted hierarchical nodes where left paths carry smaller numbers and right values hold larger scales.

Part 11 – Algorithms & Big-O Complexity

Algorithms execute logical tasks, evaluated using Big-O notation to track time and space scalability scales.

1. Searching Operations

  • Linear Search (O(n) Complexity): Scans through every record in an unorganized array sequentially.
  • Binary Search (O(log n) Complexity): Constantly cuts a sorted data list in half to find a specific key. Requires pre-sorted inputs.

2. Core Performance Complexities

  • O(1) [Constant Time]: Execution speed remains completely independent of data input volume.
  • O(n) [Linear Time]: Execution runtime scales in a direct straight line relative to the size of incoming data.
  • O(n²) [Quadratic Time]: Performance drops sharply due to nested loops running inside data logic sets.

Part 12 – Laravel Framework Core Master Class

An enterprise-level overview of Laravel components, tools, and operational setups.

1. Security Token Controls & APIs

Laravel utilizes Sanctum to deliver lightweight, secure, token-based API authentication states for single-page applications and mobile layers without bulky OAuth configurations.

2. Database Schema Migrations & Testing Seeders

  • Migrations: Code-driven table structural blueprints acting as version control tracking histories for database alterations.
  • Seeders & Factories: Scripts injected to populate tables with starting structural keys or bulk fake development data.

3. System Performance Optimizations: Jobs & Queues

Time-consuming, heavy infrastructure tasks like system alert logs or image processing operations are dispatched into Background Jobs and queued using Redis or local databases. This prevents application pages from hanging or running slowly for users.

4. Production Server Hosting Deployments

To deploy local applications to virtual private servers (like Hostinger VPS environments), you configure Nginx/Apache tracking blocks, isolate credentials inside secure .env configurations, provision active SSL encryptions via Let's Encrypt, and secure route caching through Artisan terminal commands:

php artisan config:cache
php artisan route:cache
php artisan migrate --force
